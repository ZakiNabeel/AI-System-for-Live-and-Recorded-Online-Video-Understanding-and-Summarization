# Plan 4.3 — Output Formatting

> **Self-contained scope.** Take the structured `summary.raw.json` from Plan 4.2 and the `transcript.aligned.json` / `frames.json` references, and produce the final user-facing deliverables: Markdown report, HTML report, JSON report, and a "report card" that the project assignment grades. No AI calls — just templating and file writing.

---

## 1. Objective

Build `src/output/formatter.py` that produces:

1. **`summary.md`** — human-readable Markdown with embedded image links and clickable timestamps (`[03:21]` style; if YouTube URL is known, link to `?t=201s`).
2. **`summary.html`** — same content rendered with embedded image data URIs, suitable for sharing as a single file.
3. **`report.json`** — final canonical machine-readable artifact (`Summary` + run metadata).
4. **`chapters.txt`** — YouTube-style chapter markers, ready to paste into a video description.
5. **`report_card.md`** — a status table mapping each PRD requirement to a deliverable file (used for grading / submission).

This is the last stage of the per-run pipeline. After this, `data/output/<run_id>/` contains everything a reader needs.

## 4. Phased Implementation

### Phase A — Skeleton + dataclasses (~30 min)
Create `src/output/`, `src/output/formatter.py`, `src/output/templates/`. Define `FormatResult`.

### Phase B — Helpers (~1 hr)
```python
def fmt_time(seconds: float, with_hours_if_long: float = None) -> str:
    s = int(round(seconds))
    h, rem = divmod(s, 3600); m, sec = divmod(rem, 60)
    threshold = with_hours_if_long or 0
    if seconds >= 3600 or threshold >= 3600:
        return f"{h:02d}:{m:02d}:{sec:02d}"
    return f"{m:02d}:{sec:02d}"

def yt_link(seconds: float, url: str) -> str:
    if not url: return ""
    t = int(round(seconds))
    sep = "&" if "?" in url else "?"
    return f"{url}{sep}t={t}s"

def md_timestamp(t, url):
    label = fmt_time(t)
    return f"[{label}]({yt_link(t, url)})" if url else f"`{label}`"
```

### Phase C — Markdown writer (~1 hr 30 min)
Implement `render_markdown(summary, fused, visual, ctx) -> str` filling the template. Use plain Python f-strings (or Jinja2 if the template grows complex):
```python
def render_markdown(summary, fused, visual, ctx):
    parts = [render_header(...), render_tldr(...), render_summary(...),
             render_chapters(...), render_keypoints(...),
             render_events(...), render_visuals(...), render_qa(...)]
    return "\n\n".join(p for p in parts if p)
```

For `render_visuals`, pick **at most 8** visuals to embed:
- prefer frames whose timestamp matches a `KeyPoint` or `DetectedEvent`,
- else the frames with the longest OCR text,
- always include the first frame.

### Phase D — HTML writer (~1 hr)
Two options:
1. **Pandoc** (if available on the user's machine) — best output. Detect via `shutil.which("pandoc")`.
2. **`markdown` Python package + minimal CSS** — zero external dep.

Go with option 2 as primary, option 1 as upgrade. Embedded images:
```python
def to_data_uri(path, max_width):
    img = Image.open(path); img.thumbnail((max_width, 9999))
    buf = io.BytesIO(); img.save(buf, format="PNG")
    return "data:image/png;base64," + base64.b64encode(buf.getvalue()).decode()
```

Stick the result inside `<img src="data:..." />` so the HTML is single-file.

CSS: ship a small `templates/report.css` with readable defaults (max-width 760 px, system font stack, soft borders on tables).

### Phase E — Final JSON (~30 min)
```python
def render_report_json(summary, fused, visual, ctx) -> dict:
    return {
        "run_id": ctx.run_id,
        "url": ctx.url,
        "generated_at": datetime.utcnow().isoformat() + "Z",
        "summary": asdict(summary),
        "stats": {
            "transcript_word_count": ctx.transcript_word_count,
            "frame_kept_count": len(visual.frames),
            "ocr_lines_total": sum(len(f.lines) for f in visual.frames),
            "duration_sec": fused.duration_sec,
            "events_count": len(fused.events),
        },
        "deliverables": {
            "markdown": "summary.md",
            "html": "summary.html",
            "chapters": "chapters.txt"
        }
    }
```
Write with `indent=2, sort_keys=False, default=str`.

### Phase F — Chapters writer (~15 min)
YouTube convention: one chapter per line, format `MM:SS Title` (or `HH:MM:SS Title`). The first chapter must be `00:00`.

```python
def render_chapters_txt(chapters):
    lines = []
    for c in chapters:
        lines.append(f"{fmt_time(c.t_start, with_hours_if_long=last.t_end)} {c.title}")
    return "\n".join(lines)
```

### Phase G — Report card (~1 hr)
Map PRD requirements to produced files:

```markdown
# Submission Report Card

| Requirement (from PRD) | Deliverable | Path | Present |
|---|---|---|---|
| Live video analysis | Live capture log + summary | `logs/<id>.log`, `summary.md` | ✅ |
| YouTube video analysis | Pipeline output | `summary.md`, `report.json` | ✅ |
| Speech understanding | Transcript | `transcript.aligned.json`, `transcript.srt` | ✅ |
| Visual understanding | Frames + OCR | `frames/*.png`, `visual.json` | ✅ |
| Multimodal summarization | Final summary | `summary.md`, `summary.html` | ✅ |
| Time-stamped insights | Key points list | `summary.md` (Key Points section) | ✅ |
```

The script verifies each path exists (`presence: ✅` or `❌`) and writes the table.

### Phase H — CLI + main format_outputs() (~30 min)
Compose all renderers; write all files; return `FormatResult`.

### Phase I — Tests (~1 hr 30 min)

1. **Time formatter** — assert `fmt_time(0)=="00:00"`, `fmt_time(3661)=="01:01:01"`, `fmt_time(125)=="02:05"`.
2. **YouTube link builder** — handles `?` already in URL.
3. **End-to-end on a tiny fixture** — provide hand-written tiny `summary.raw.json` + `fused.json` + `visual.json`; run `format_outputs`; assert all five files exist; HTML opens in a browser without script errors (parse with `html.parser`).
4. **Image data URI** — assert HTML contains `data:image/png;base64,` prefix when `embed_images=True`; otherwise contains a relative `frames/...` path.
5. **No-YouTube mode** — timestamps render as code spans, not links.
6. **Report card** — patch one expected file to be missing; assert that row is `❌`.
7. **Unicode** — title/summary contain non-ASCII; assert UTF-8 written correctly to all outputs.

## 8. Edge Cases & Pitfalls

1. **Empty summary** — render the page anyway with explicit `(no content detected)` placeholders, so the report card still gets generated.
2. **Very long summaries** — truncate `full_summary` to 8 K characters in HTML to avoid huge pages, link to JSON for full text.
3. **Frame paths absolute on Windows** — convert all references to *relative* paths (relative to the output dir) before rendering Markdown.
4. **HTML XSS** — escape any LLM-produced text that goes into the HTML (`html.escape`); never render raw HTML from the model output.
5. **YouTube timestamp links** — only attach to URLs that look like a YouTube video URL; otherwise plain code-span. Use a regex: `r"(youtube\.com/watch\?v=|youtu\.be/)([\w-]+)"`.
6. **Pandoc presence** — if available, optionally generate `summary.pdf` via `pandoc summary.md -o summary.pdf`. Mark this deliverable optional.
7. **Image too large to embed** — resize before encoding (`Pillow.thumbnail`).
8. **First chapter not at 00:00** — prepend a synthetic `00:00 Introduction` chapter and log a warning.
9. **Filenames collisions** — outputs always go into the run-specific dir; no global names.
